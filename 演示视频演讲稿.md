# 程序演示视频——演讲稿与操作步骤

> 预计时长：10-12 分钟 | 先操作后解说

---

## 步骤 1：启动与登录 [0:00-1:00]

**操作**：双击运行 → 弹出登录窗口 → 输入 root / root123 → Login

**演讲稿**：
"大家好，这是我开发的多用户多级目录文件系统。首先以 root 身份登录。登录后标题栏显示 UID=0，角色为 root。左侧是目录树，右侧是文件列表，底部有命令行和状态栏。"

---

## 步骤 2：系统总览 [1:00-2:00]

**操作**：展开左侧目录树 → / /home /shared → 指向菜单栏和工具栏

**演讲稿**：
"根目录下有两个系统目录：home 存放用户主目录，shared 是共享目录。菜单栏有 File、View、User、Help。工具栏提供 Home、Up、New File、New Dir、Delete、Refresh、Disk 七个按钮。"

---

## 步骤 3：创建文件并编辑 [2:00-3:30]

**操作**：
1. 双击进入 /shared
2. 右键空白处 → New File → 输入 `readme`
3. 双击 readme → 打开编辑器
4. 输入 "Welcome to shared folder. Visible to all users."
5. Ctrl+S 保存

**演讲稿**：
"右键 New File 创建文件，双击打开编辑器。Ctrl+S 保存后，底层调用 writeFile 按块写入磁盘，每块 64 字节，由 FAT 表链连接。"

---

## 步骤 4：文件操作 [3:30-5:00]

**操作**：
1. 右键 readme → Copy → 弹窗确认
2. 进入 /home/admin/ → 右键 → Paste
3. 右键 readme → Rename → 输入 `info`
4. 右键 info → Change Attributes → 勾选 Read-Only → OK
5. 双击 info → 编辑 → Ctrl+S → 弹窗 "read-only file"
6. 右键 info → Properties → 查看文件信息

**演讲稿**：
"跨目录复制粘贴。重命名功能。属性修改后设为只读，再次编辑会提示只读文件。Properties 显示文件名、类型、大小、所有者和起始块号。"

---

## 步骤 5：磁盘可视化 [5:00-5:30]

**操作**：点击工具栏 Disk 按钮

**演讲稿**：
"128 个物理块的可视化。橙色是 FAT 表，蓝色根目录，绿色空闲，红色和浅橙色是已使用块。可以看到文件占用的数据块链。"

---

## 步骤 6：注册新用户 [5:30-6:30]

**操作**：
1. User → Logout → 弹出登录窗口
2. 点击 Register → 输入 user2 / 222222 → 注册成功
3. 用 user2 登录

**演讲稿**：
"演示多用户功能。注册 user2，系统自动创建 /home/user2。用户数据持久化到 users.dat，重启不丢失。标题栏显示 UID=2。"

---

## 步骤 7：权限隔离 [6:30-8:00]

**操作**：
1. user2 登录后，展开 /home/ → 只看到 user2
2. 命令行输入 `cd /home/admin` → 弹窗 "access denied"
3. 进入 /shared/ → 能看到 root 创建的 readme
4. 在 /shared/ 创建文件 `user2_share`
5. Logout → 以 admin 登录
6. admin 进入 /shared/ → 能看到 user2 创建的文件

**演讲稿**：
"权限隔离的核心。user2 在 home 下只能看到自己的目录。命令行进入他人目录被拒绝。但 shared 是共享目录，所有用户可见全部文件。三层过滤确保权限一致。"

---

## 步骤 8：Cut/Paste [8:00-9:00]

**操作**：
1. admin 在 /home/admin/ 新建 `test`，写入内容
2. 右键 test → Cut → 弹窗确认
3. 进入 /shared/ → 右键 Paste → 弹窗 "Moved"
4. 回到 /home/admin/ → test 已消失

**演讲稿**：
"Cut 存入剪贴板，Paste 读取源内容写入目标目录并删除源文件，实现了文件的跨目录移动。"

---

## 步骤 9：root vs admin 权限 [9:00-10:00]

**操作**：
1. admin 登录 → 进入 /home/user1/ → 右键删除普通用户文件 → 成功
2. Logout → root 登录 → 在根目录创建 `sysfile`
3. Logout → admin 登录 → 右键 sysfile → Delete → "permission denied"
4. admin 点击 File → Format → "only root can format disk"

**演讲稿**：
"root 和 admin 的关键区别。admin 可以管理普通用户，但不能删除 root 创建的文件——owner=0 受保护。只有 root 能格式化磁盘。"

---

## 步骤 10：命令行与帮助 [10:00-11:00]

**操作**：
1. 命令行输入 `help` → 弹窗
2. `dir` → 列出当前目录
3. `mkdir demo` → `cd demo` → `create hello`
4. Help → System Help → 查看完整帮助

**演讲稿**：
"系统也支持命令行。help 列出所有命令。System Help 提供完整的架构概述和命令参考。"

---

## 步骤 11：数据持久化 [11:00-11:30]

**操作**：
1. File → Save Disk → 关闭程序
2. 重新启动 → admin 登录 → 进入 /shared/ → 文件都在
3. User → Users → user2 仍然存在

**演讲稿**：
"disk.img 保存磁盘数据，users.dat 保存用户账号。重启后全部保留。"

---

## 步骤 12：结束 [11:30-12:00]

**演讲稿**：
"以上就是多用户多级目录文件系统的全部演示。项目采用四层架构，16 个源文件，2400 多行代码，实现了文档全部 13 项命令，并扩展了复制粘贴、重命名、属性修改、四级权限和共享目录功能。谢谢大家。"
